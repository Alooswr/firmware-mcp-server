# 安全策略

[English](SECURITY.md)

## 支持版本

`main` 分支会接收安全修复。当前项目暂未维护独立的长期支持发布分支。

## 漏洞报告

如果怀疑存在安全漏洞，请不要直接创建公开 issue。

请通过 GitHub 联系方式私下报告给仓库所有者，并尽量包含：

- 问题的简短说明。
- 最小复现步骤或概念验证代码，如果有的话。
- 受影响的操作系统和 Python 版本。
- 问题是否依赖恶意 `devices.json`，还是仅通过 MCP 客户端即可触发。

## 运行安全建议

Firmware MCP Server 是一个本地自动化桥接程序。它会按照用户配置执行本地命令，因此设备配置应被视为可信输入。

建议做法：

- 保持 `config/devices.json` 私有并仅用于本机。
- 连接 MCP 客户端前，审查配置中的构建、烧录、复位和串口设置。
- 避免把命令指向不可信仓库中的脚本或二进制文件。
- 为构建输出和日志使用专门的工作目录。
- 使用设备工具所需的最低权限运行服务器。
- 不要通过未认证的网络包装层暴露 stdio 传输。

内置安全属性：

- 服务器不启动 HTTP 监听。
- 外部命令执行不使用 `shell=True`。
- 本机设备配置默认被 git 忽略。
- stdout 保留给 MCP 协议通信，降低协议输出被意外污染的风险。
